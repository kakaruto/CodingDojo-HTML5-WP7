jQuery Mobile

Qu'est-ce que jQuery Mobile ?

  • Framework Mobile utilisant jQuery (bien évidemment...).
  • Cross-browser, cross-platform.
  • Création d'application web qui ressemble au maximum à du natif.
  • Markup driven, peu de code requis.

Platformes cibles

  • iOS, Android, Windows Mobile
  • Blackberry, Symbian, webOS
  • Firefox Mobile (Fennec), Opera Mobile / Mini
  • Meego, bada, Maemo...
  • Phonegap!

Elements de page basiques

  • Les éléments sont déclarés avec des DIVs qui ont des attributes 'data-role'.
  • Au moins un élément 'data-role="page"'.
  • Dans la page, au moins un élément 'data-role="content"'.
  • Les classes CSS sont ajoutées automatiquement.
  • Les formulaires et les listes sont automatiquement améliorés.

Page basique

<!doctype html> 
<html> 
  <head> 
  <title>Page Title</title> 
  <link rel="stylesheet" href="jquery.mobile.css" />
  <script src="jquery.js"></script>
  <script src="jquery.mobile.js"></script>
</head> 
<body> 

<div data-role="page">

  <div data-role="header">
    <h1>Page Title</h1>
  </div>

  <div data-role="content">  
    <p>Page content goes here.</p>    
  </div>

  <div data-role="footer">
    <h4>Page Footer</h4>
  </div>
</div>

</body>
</html>

Chargement AJAX

  • Chaque page doit respecter le standard jQuery Mobile.
  • Les pages sont injectées dans le DOM si AJAX est supporté, sinon un chargement classique se produit.
  • Si le chargement de la page échouse, l'utilisateur recoit un "joli" message d'erreur.
  • Quand on fait un lien vers une page non-mobile sur le même domaine, il faut utiliser rel="external".
  • Les liens vers d'autres domaines sont automatiquement détectés, générant un chargement classique.

Focus sur le jeu mobile

  • la balise audio est mal supportée
  • canvas est lent
  • HTML/CSS "est acceléré"
  • Il faut utiliser les évenements Touch et pas Click
  • Le moteur JS est plus lent, attention aux gros calcul mathématique

Comment dessiner pour le jeu ?

  • SVG
  • Balises HTML
  • Canvas

SVG

  • Pour dessiner en vectoriel
  • l'arbre des objets est conservé en mémoire
  • Acces au SVG via le DOM
  • CSS applicable

Exemple SVG

<svg width="400" height="200" xmlns="http://www.w3.org/2000/svg">
    <rect fill="red" x="20" y="20" width="100" height="75" />
    <rect fill="blue" x="50" y="50" width="100" height="75" />
</svg>

Balises HTML

  • Balises classiques
  • Haut niveau, peut réduire les cycles de développements
  • Peut dans certains scénarios, être plus performant que le canvas (mobile et iOs)

Canvas

  • Dessin dans un bitmap
  • API Javascript
  • Primitives de dessin
  • Rectangles, lignes, courbes, …
  • Juste du dessin, la gestion des objets est à notre charge
  • Contenu non visible dans le DOM

Exemple canvas

<canvas id="myCanvas" width="200" height="200">
  Votre navigateur ne supporte pas l’élément canvas, désolé.
</canvas>

<script type="text/javascript">
  var example = document.getElementById("myCanvas"); 
  var context = example.getContext("2d"); 
  context.fillStyle = "rgb(255,0,0)"; 
  context.fillRect(30, 30, 50, 50); 
</script>
